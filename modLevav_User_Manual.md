## א. סקירה כללית

מערכת **modLevav** היא מודול VBA המותקן בקובץ Excel, ומיועד לניהול, ביקורת והשוואת נתוני ביטוח בין שנים. המערכת מבצעת שלושה תהליכים עיקריים:

**כפתור 1 — בדיקת נתונים:** קריאת קובץ מקור של שנה נבחרת, זיהוי חריגות ושגיאות, ויצירת גיליון ביקורת ("לטיפול") שבו ניתן לתקן את הבעיות.

**כפתור 2 — יישום ודו"חות:** יישום התיקונים מגיליון הביקורת, בניית גיליונות בסיס (נקיים) לשתי השנים, ויצירת גיליונות השוואה לפי חתכים שונים — חודשים, חברות, ענף מרכז, טלריות, וסוכנים.

**כפתור 3 — ייצור מצגת:** יצירת מצגת PowerPoint אוטומטית מנתוני ההשוואה, כולל גרפים וטבלאות.

## ג. התקנה ראשונית

### שלב א' — ייבוא המודול

א. פתח את קובץ ה-Excel.  
ב. לחץ `Alt + F11` כדי לפתוח את עורך ה-VBA.  
ג. בתפריט **File** → **Import File** → בחר את הקובץ `modLevav_v6.bas`.  
ד. ודא שהמודול מופיע תחת **Modules** בפאנל השמאלי.  

> **חשוב:** אם קיים מודול ישן בשם `modLevav` או `Module1`, מחק אותו קודם (קליק ימני → Remove).

### שלב ב' — קוד דף הבית

א. בעורך ה-VBA, מצא את גיליון **"דף הבית"** בפאנל השמאלי.  
ב. לחץ עליו פעמיים כדי לפתוח את חלון הקוד שלו.  
ג. הדבק את הקוד הבא:

```vba
Private Sub Worksheet_Change(ByVal Target As Range)
    If Target.Address = "$B$4" Then
        Application.EnableEvents = False
        UpdatePeriodDropdown
        Application.EnableEvents = True
    End If
End Sub
```

ד. שמור את הקובץ כ-`.xlsm` (Excel Macro-Enabled Workbook).

### שלב ג' — הגדרה ראשונית

א. עבור לגיליון **"דף הבית"**.  
ב. הקלד בעורך ה-VBA בחלון ה-Immediate (Ctrl+G):  
```
SetupMainSheet
```
ג. לחץ Enter. שלושה כפתורים יופיעו בדף הבית, וגיליון ההגדרות יאותחל.

## ה. הגדרת פרמטרים

### הכנת קבצי מקור מתוכנת עתיד

לפני כל הרצה, יש להכין את קבצי הנתונים מתוכנת עתיד:

א. **שליפת הנתונים:** בתוכנת עתיד, בצע שליפה של **חתך פוליסות** לשנה הרלוונטית.  
ב. **שמירת הקובץ:** שמור את קובץ השליפה בשם **XXXX.xls** כאשר XXXX הוא מספר השנה (לדוגמה: `2025.xls`).  
ג. **מיקום הקובץ:** העבר את הקובץ לתיקייה `C:\פרויקט לבב\SOURCE\`.  
ד. **חזור על התהליך** עבור כל שנה שתרצה להשוות (שנת בסיס ושנה נבדקת).

**דוגמה:** להשוואת 2024 מול 2025:
   - `C:\פרויקט לבב\SOURCE\2024.xls`
   - `C:\פרויקט לבב\SOURCE\2025.xls`

> **הערה:** המערכת תומכת גם בפורמט `.xlsx` וגם בפורמט `.xls` — אין צורך להמיר את הקובץ.

### לפני הרצה ראשונה

ב. **הגדרת שנים:** בדף הבית, הזן:
   - **B2** = שנת הבסיס (לדוגמה: `2024`)
   - **B3** = השנה הנבדקת (לדוגמה: `2025`)

ג. **בחירת תקופה:** בתא **B4** בחר את סוג התקופה מהרשימה הנפתחת:
   - **שנתי** — השוואה שנתית מלאה (ברירת מחדל)
   - **חצי שנתי** — בחר ב-E4 מחצית ראשונה/שנייה
   - **רבעוני** — בחר ב-E4 את הרבעון
   - **חודשי** — בחר ב-E4 את החודש

ד. **עמודת תאריך (B5):** בחר לפי איזה תאריך לסנן — "תחילת ביטוח" או "בורדרו".

### גיליון הגדרות (ניהול)

גיליון **"הגדרות"** מכיל טבלאות הגדרה שמשפיעות על תהליך הביקורת:

| עמודות | תפקיד |
|--------|--------|
| E-F | טבלת שדות — שם השדה בעברית ומספר העמודה בקובץ המקור |
| G | סוג בדיקה לכל שדה |
| H | האם השדה הוא מפתח |
| J-K | טבלת פרמטרים (לדוגמה: סף פרמיה) |
| N-O | טבלת תרגום (מיפוי ענפים לענף מרכז) |

## ו. כפתור 1 — בדיקת נתונים (BuildReview)

### מה הוא עושה

א. פותח את קובץ המקור של השנה הנבדקת (B3) מתיקיית `C:\פרויקט לבב\SOURCE\`.  
ב. קורא את הגדרות הבדיקה מגיליון ההגדרות.  
ג. עובר שורה-שורה על הנתונים ובודק:
   - שדות חסרים (לפי הגדרת הבדיקה)
   - חריגות פרמיה (מעל הסף שהוגדר)
   - שדות מפתח ריקים  

ד. יוצר גיליון **"לטיפול_{שנה}"** עם כל השורות הבעייתיות.  
ה. מוסיף עמודת **פעולה** (רשימה נפתחת: תקן / התעלם / מחק) ועמודת **תיקון** (טקסט חופשי).

### הודעות

- **"הסתיים — נמצאו X חריגות"** — הביקורת הסתיימה, יש X שורות לטיפול.
- **"הסתיים בהצלחה"** — לא נמצאו חריגות.

### מה לעשות אחרי

עבור לגיליון **"לטיפול_{שנה}"** ותקן את הבעיות (ראה פרק יא).

## ז. כפתור 2 — יישום ודו"חות (ApplyCorrectionsAndBuildReports)

### תנאי הפעלה

לפני הלחיצה, ודא:  
א. הרצת כפתור 1 קודם (אם יש צורך בביקורת).  
ב. טיפלת בשורות הבעייתיות בגיליון "לטיפול".  
ג. קבצי המקור של **שתי השנים** (B2 ו-B3) נמצאים ב-`C:\פרויקט לבב\SOURCE\`.

### מה הוא עושה

**שלב א' — בניית בסיס שנה נבדקת:**  
א. קורא את קובץ המקור של השנה הנבדקת.  
ב. מיישם את התיקונים מגיליון "לטיפול" (שינוי ערכים, מחיקת שורות, התעלמות).  
ג. מסנן לפי סף פרמיה ותקופה.  
ד. יוצר גיליון **"בסיס_{שנה}"** עם הנתונים הנקיים.

**שלב ב' — בניית בסיס שנת ייחוס:**  
א. אם כבר קיים גיליון **"בסיס_{שנת ייחוס}"** — משתמש בו כמות שהוא (כולל תיקונים קודמים).  
ב. אם לא קיים — בונה אותו מקובץ המקור של שנת הייחוס.

**שלב ג' — בניית גיליונות השוואה:**  
יוצר גיליונות השוואה לפי חתכים:

| גיליון | חתך |
|---------|------|
| **חודשים** | השוואה חודש-חודש |
| **חברות** | השוואה לפי חברת ביטוח |
| **ענף מרכז** | השוואה לפי ענף ראשי |
| **טלרים** | השוואה לפי טלר/ית |
| **סוכנים** | השוואה לפי סוכן |

### מבנה גיליון השוואה

כל גיליון השוואה מכיל את העמודות הבאות:

| עמודה | תוכן |
|--------|--------|
| A | שם הקטגוריה (חברה/סוכן/חודש וכו') |
| B-D | **פרודוקציה (פרמיות):** שנת ייחוס, שנה נבדקת, שינוי% |
| E-G | **מסמכים:** שנת ייחוס, שנה נבדקת, שינוי% |
| H-J | **מבוטחים:** שנת ייחוס, שנה נבדקת, שינוי% |
| K-M | **פוליסות:** שנת ייחוס, שנה נבדקת, שינוי% |
| N-P | **עמלה:** שנת ייחוס, שנה נבדקת, שינוי% |

השורה האחרונה בכל גיליון היא **שורת סיכום** עם סכומים כוללים.

ב. PowerPoint מותקן במחשב (גרסת Desktop).

### מה הוא עושה

## ט. גיליון הגדרות (ניהול)

עמודה S מכילה את כל הטקסטים של הודעות המערכת בעברית. **אין לשנות עמודה זו** — היא מתעדכנת אוטומטית בהרצת `SetupMainSheet`.

## י. גיליונות ההשוואה

### חישוב מבוטחים

ספירת מבוטחים מתבצעת לפי **מספרי לקוח ייחודיים** (לא כפולים), תוך **סינון ביטולים** — שורות שבהן עמודת הפעולה מכילה את המילה "ביטול" לא נספרות.

### חישוב שינוי באחוזים

עמודת השינוי מחושבת כ:  
**(ערך שנה נבדקת − ערך שנת ייחוס) / ערך שנת ייחוס × 100**

אם ערך שנת הייחוס הוא 0, עמודת השינוי תציג 0.

### שורת סיכום

השורה האחרונה בכל גיליון מכילה סכומים כוללים. לעמודות מבוטחים ופוליסות, הסיכום הוא **ספירה ייחודית כוללת** (לא סכום של השורות — כי מבוטח יכול להופיע ביותר מקטגוריה אחת).

## יא. גיליון לטיפול (ביקורת)

### מבנה

גיליון **"לטיפול_{שנה}"** מכיל:
- **עמודה A:** מספר שורה בקובץ המקור
- **עמודות B ואילך:** השדות שהוגדרו בטבלת השדות
- **עמודת סיבה:** קוד הסיבה לחריגה
- **עמודת פעולה:** רשימה נפתחת עם האפשרויות:
  - **תקן** — השורה תיכלל בבסיס עם התיקון שהוזן
  - **התעלם** — השורה תוסר מהבסיס
  - **מחק** — השורה תוסר מהבסיס
  - **(ריק)** — השורה תיכלל בבסיס כמות שהיא
- **עמודת תיקון:** טקסט חופשי — הערך המתוקן

### איך לתקן

א. עבור לגיליון "לטיפול".  
ב. לכל שורה בעייתית, בחר פעולה מהרשימה הנפתחת.  
ג. אם בחרת "תקן", הזן את הערך הנכון בעמודת התיקון.  
ד. חזור לדף הבית והרץ כפתור 2.

> **טיפ:** ניתן להריץ כפתור 1 מספר פעמים — כל הרצה יוצרת גיליון "לטיפול" חדש. תיקונים שנעשו בגיליון הקודם יישמרו אם לא מחקת אותו.

## יב. פתרון בעיות

### שגיאות נפוצות

| שגיאה | סיבה | פתרון |
|--------|--------|--------|
| "REF SOURCE NOT FOUND" | קובץ המקור לא נמצא | ודא שהקובץ `{שנה}.xls` נמצא ב-`C:\פרויקט לבב\SOURCE\` |

### הודעת שגיאה כללית

כל הודעת שגיאה מציגה:

- **Err #:** מספר השגיאה
- **Step:** השלב בתהליך
- **Description:** תיאור השגיאה

שמור את המידע הזה לצורך דיווח ותיקון.

## יג. גיבוי ושחזור

### גיליון בסיס

גיליון **"בסיס_{שנה}"** נשמר בקובץ ה-Excel. אם הוא קיים, המערכת משתמשת בו מחדש (כולל תיקונים שנעשו) במקום לבנות אותו מחדש. 

**לאלץ בנייה מחדש:** מחק את גיליון הבסיס הרלוונטי לפני הרצת כפתור 2.

### גיבוי הקוד

הקוד מגובה ב-GitHub בריפוזיטורי פרטי. כל גרסה מתויגת עם מספר גרסה ותיאור השינויים.

### סדר עבודה מומלץ

א. הכן את קבצי המקור מתוכנת עתיד (חתך פוליסות) ושמור ב-`C:\פרויקט לבב\SOURCE\` בשם השנה (לדוגמה: `2025.xls`).  
ב. הגדר שנים ותקופה בדף הבית.  
ג. הרץ **כפתור 1** — בדוק את הנתונים.  
ד. תקן בעיות בגיליון "לטיפול".  
ה. הרץ **כפתור 2** — ייצר דוחות.  
ו. בדוק את גיליונות ההשוואה.  
ז. הרץ **כפתור 3** — ייצר מצגת.  
ח. פתח את המצגת ובדוק.
